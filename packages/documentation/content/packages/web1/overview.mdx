# @moneyworks/web1

A modern, type-safe web application for MoneyWorks built with React Router v7, React Query, and Radix UI. Features multi-tenant connection management, secure credential storage, and a beautiful UI powered by Tailwind CSS and shadcn/ui components.

## Quick Start

### Installation

```bash
bun add @moneyworks/web1
```

### Development

```bash
# Start development server
bun dev

# Type checking
bun typecheck

# Build for production
bun build

# Start production server
bun start
```

### Environment Variables

```bash
# Required
VITE_CLERK_PUBLISHABLE_KEY=pk_test_...

# Optional
VITE_AUTOMATION=true  # Bypass auth for testing
```

#### MoneyWorks Integration
```typescript
import { useMoneyWorks } from "~/hooks/use-moneyworks";

function TaxRatesList() {
  const { client } = useMoneyWorks();
  
  const { data, isLoading } = useQuery({
    queryKey: ['taxRates'],
    queryFn: async () => {
      if (!client) throw new Error('No connection');
      return client.export('TaxRate', { format: 'full' });
    },
    enabled: !!client
  });
  
  if (isLoading) return <Skeleton />;
  
  return <TaxRatesTable data={data} />;
}
```

### Actions

Handle form submissions and mutations:

```typescript
export async function action({ request }: ActionFunctionArgs) {
  const user = await requireAuth(request);
  const formData = await request.formData();
  
  const result = await createConnection({
    userId: user.id,
    name: formData.get("name"),
    host: formData.get("host"),
    // ... other fields
  });
  
  if (result.success) {
    return redirect("/connections");
  }
  
  return { errors: result.errors };
}
```

## State Management

### React Query Configuration

```typescript
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 1000 * 60 * 5, // 5 minutes
      gcTime: 1000 * 60 * 10, // 10 minutes
      retry: 3,
      retryDelay: attemptIndex => Math.min(1000 * 2 ** attemptIndex, 30000),
    },
  },
});
```

### Connection Context

Global connection state management:

```typescript
export const ConnectionProvider = ({ children }) => {
  const [currentConnectionId, setCurrentConnectionId] = useState<string | null>();
  const { data: connections } = useConnections();
  
  const currentConnection = useMemo(() => 
    connections?.find(c => c.id === currentConnectionId),
    [connections, currentConnectionId]
  );
  
  const switchConnection = useCallback((id: string) => {
    setCurrentConnectionId(id);
    queryClient.invalidateQueries();
  }, []);
  
  return (
    <ConnectionContext.Provider value={{
      connections,
      currentConnection,
      switchConnection,
      // ... other methods
    }}>
      {children}
    </ConnectionContext.Provider>
  );
};
```

## Database Schema

### SQLite with Drizzle ORM

```typescript
// Connections table
export const connections = sqliteTable("connections", {
  id: text("id").primaryKey(),
  user_id: text("user_id").notNull(),
  name: text("name").notNull(),
  
  // MoneyWorks connection details
  mw_host: text("mw_host").notNull(),
  mw_port: integer("mw_port").notNull(),
  mw_data_file: text("mw_data_file").notNull(),
  mw_username: text("mw_username").notNull(),
  mw_password: text("mw_password").notNull(), // Encrypted
  
  // Optional folder auth
  mw_folder_name: text("mw_folder_name"),
  mw_folder_password: text("mw_folder_password"), // Encrypted
  
  // Metadata
  is_active: integer("is_active").default(1),
  created_at: integer("created_at").notNull(),
  updated_at: integer("updated_at").notNull(),
});

// Groups for organizing connections
export const groups = sqliteTable("groups", {
  id: text("id").primaryKey(),
  user_id: text("user_id").notNull(),
  name: text("name").notNull(),
  parent_id: text("parent_id"),
  created_at: integer("created_at").notNull(),
});

// Group members (many-to-many)
export const group_members = sqliteTable("group_members", {
  group_id: text("group_id").notNull(),
  connection_id: text("connection_id").notNull(),
  added_at: integer("added_at").notNull(),
});

// Audit logs
export const audit_logs = sqliteTable("audit_logs", {
  id: text("id").primaryKey(),
  user_id: text("user_id").notNull(),
  connection_id: text("connection_id"),
  action: text("action").notNull(),
  details: text("details"), // JSON
  ip_address: text("ip_address"),
  created_at: integer("created_at").notNull(),
});
```

## Best Practices

### 1. Connection Management
- Always check for active connection before MoneyWorks operations
- Handle connection errors gracefully
- Implement connection health checks
- Cache connection status

### 2. Performance
- Use React Query for all data fetching
- Implement optimistic updates for better UX
- Lazy load heavy components
- Prefetch data on route transitions

### 3. Security
- Never expose encryption keys
- Validate all user inputs
- Implement rate limiting on API routes
- Use HTTPS in production
- Regular security audits

### 4. Error Handling
- Provide clear error messages
- Log errors for debugging
- Implement error boundaries
- Show fallback UI for errors

### 5. Accessibility
- Test with screen readers
- Ensure keyboard navigation works
- Provide proper ARIA labels
- Maintain color contrast ratios
