# @moneyworks/api

A comprehensive REST API for MoneyWorks accounting software, built with ElysiaJS for blazing-fast performance and exceptional developer experience. The API provides type-safe access to MoneyWorks data with progressive enhancement - new tables are added as they're vetted and tested.

## Quick Start

### Installation

```bash
bun add @moneyworks/api
```

### Basic Usage

```typescript
import { createApp } from '@moneyworks/api/app';
import { createSmartClient, loadConfig } from '@moneyworks/data';

// Load MoneyWorks configuration
const mwConfig = await loadConfig('./mw-config.json');

// Create MoneyWorks client
const client = createSmartClient(mwConfig);

// Create and start API server
const app = createApp(client, {
  port: 3000,
  enableSwagger: true,
  enableCors: true
});

app.listen({ port: 3000, hostname: '0.0.0.0' });
```

### Docker Deployment

```dockerfile
FROM oven/bun:1-alpine
WORKDIR /app
COPY . .
RUN bun install --production
EXPOSE 3000
CMD ["bun", "run", "start"]
```

### 📊 Multiple Export Formats
All table endpoints support multiple formats via the `format` query parameter:

```typescript
// Full objects (default) - Best for development
GET /api/v1/tables/TaxRate?format=full
[
  { "Code": "GST10", "Description": "GST 10%", "Rate": 0.1 }
]

// Compact arrays - Minimal network overhead
GET /api/v1/tables/TaxRate?format=compact
[
  ["GST10", "GST 10%", 0.1]
]

// Compact with headers - Self-documenting arrays
GET /api/v1/tables/TaxRate?format=compact-headers
[
  ["Code", "Description", "Rate"],
  ["GST10", "GST 10%", 0.1]
]

// Schema-enriched - Complete metadata
GET /api/v1/tables/TaxRate?format=schema
{
  "data": [...],
  "schema": {
    "fields": {
      "Code": { "type": "string", "maxLength": 8 },
      "Description": { "type": "string", "maxLength": 100 },
      "Rate": { "type": "number", "precision": 6 }
    }
  }
}
```

## Architecture

### Layered Design

```
┌─────────────────────────────────────┐
│           API Routes                │
│  (Tables, Eval, Company, I18n)      │
├─────────────────────────────────────┤
│          Controllers                │
│  (Business Logic, Validation)       │
├─────────────────────────────────────┤
│          Middleware                 │
│  (Auth, Logging, Errors, i18n)      │
├─────────────────────────────────────┤
│         Table Registry              │
│  (Progressive Enhancement)          │
├─────────────────────────────────────┤
│      MoneyWorks Client              │
│     (@moneyworks/data)              │
└─────────────────────────────────────┘
```

### Request Flow

1. **Request arrives** → Request ID assigned
2. **Middleware pipeline** → Auth, i18n, logging
3. **Route handler** → Parameter validation
4. **Controller** → Business logic
5. **MoneyWorks client** → Data access
6. **Response formatting** → Based on requested format
7. **Error handling** → Consistent error format

## Advanced Usage

### Custom Table Controllers

Extend the API with custom table controllers:

```typescript
import { BaseTableController } from '@moneyworks/api/controllers/base-table';
import type { SmartMoneyWorksClient } from '@moneyworks/data';

export class AccountController extends BaseTableController {
  constructor(client: SmartMoneyWorksClient) {
    super(client, 'Account');
  }
  
  // Add custom business logic
  async getChartOfAccounts() {
    const accounts = await this.export({
      filter: 'Active = true',
      orderBy: 'Code'
    });
    
    // Build hierarchical structure
    return this.buildAccountTree(accounts);
  }
}
```

### Middleware Customization

Add custom middleware for specific needs:

```typescript
import { Elysia } from 'elysia';

// API key authentication
const apiKeyAuth = new Elysia()
  .derive(({ headers }) => {
    const apiKey = headers['x-api-key'];
    if (!apiKey || !isValidApiKey(apiKey)) {
      throw new Error('Invalid API key');
    }
    return { apiKey };
  });

// Apply to app
const app = createApp(client)
  .use(apiKeyAuth);
```

### Error Handling

Consistent error responses across all endpoints:

```typescript
// All errors follow this format
{
  "error": {
    "code": "TABLE_NOT_FOUND",
    "message": "Table Account is not yet available",
    "details": {
      "available": ["TaxRate"],
      "requested": "Account"
    },
    "requestId": "mw-12345-abcdef"
  }
}

// Custom error handling
app.onError(({ code, error, set }) => {
  if (error instanceof MoneyWorksError) {
    set.status = 400;
    return {
      error: {
        code: 'MW_ERROR',
        message: error.message,
        details: error.details,
        requestId: context.requestId
      }
    };
  }
});
```

## Best Practices

### 1. Use Appropriate Export Formats
- Development: Use `full` format for readability
- Production: Use `compact` for minimal overhead
- Data pipelines: Use `compact-headers` for self-documenting arrays
- Documentation: Use `schema` format for complete metadata

### 2. Implement Pagination
Always paginate large result sets:

```typescript
GET /api/v1/tables/Transaction?limit=100&offset=0
```

### 3. Filter at the Source
Use MoneyWorks filters to reduce data transfer:

```typescript
GET /api/v1/tables/TaxRate?filter=Rate>0
```

### 4. Cache Strategically
- Cache company information (changes rarely)
- Cache field labels (static per version)
- Don't cache transactional data

### 5. Handle Errors Gracefully
Always check for error responses and handle appropriately:

```typescript
const response = await fetch('/api/v1/tables/TaxRate');
const data = await response.json();

if ('error' in data) {
  console.error(`API Error: ${data.error.code} - ${data.error.message}`);
  return;
}

// Process successful response
```

## API Endpoints Reference

- `GET /api/v1/tables` - List available tables

- `POST /api/v1/eval` - Evaluate expression

### Company Information
- `GET /api/v1/company` - Get company info
