# API Summary

Quick reference guide for all exports from @moneyworks/utilities.

## Core Functions

### Date Functions (YYYYMMDD)

| Function | Description | Example |
|----------|-------------|---------|
| `d` | Tagged template for dates with fluent API | `d\`20250115\`.addDays(30)` |
| `isYYYYMMDD` | Type guard for YYYYMMDD strings | `if (isYYYYMMDD(value)) { ... }` |
| `createYYYYMMDD` | Create YYYYMMDD with validation | `createYYYYMMDD("20250115")` |
| `tryCreateYYYYMMDD` | Safe creation (returns null on invalid) | `tryCreateYYYYMMDD(input)` |
| `parseToYYYYMMDD` | Parse various formats to YYYYMMDD | `parseToYYYYMMDD("2025-01-15")` |
| `todayYYYYMMDD` | Get today's date | `todayYYYYMMDD()` |
| `addDaysToYYYYMMDD` | Add days to date | `addDaysToYYYYMMDD(date, 30)` |
| `addMonthsToYYYYMMDD` | Add months to date | `addMonthsToYYYYMMDD(date, 3)` |
| `daysBetweenYYYYMMDD` | Calculate days between | `daysBetweenYYYYMMDD(from, to)` |
| `compareYYYYMMDD` | Compare dates (-1, 0, 1) | `dates.sort(compareYYYYMMDD)` |
| `formatYYYYMMDD` | Format with separator | `formatYYYYMMDD(date, "/")` |
| `startOfMonthYYYYMMDD` | First day of month | `startOfMonthYYYYMMDD(date)` |
| `endOfMonthYYYYMMDD` | Last day of month | `endOfMonthYYYYMMDD(date)` |

### Period Functions

| `currentPeriod` | Get current period | `currentPeriod()` |
| `periodYear` | Extract year | `periodYear(period)` |
| `periodMonth` | Extract month | `periodMonth(period)` |

| `monthsBetweenPeriods` | Calculate months | `monthsBetweenPeriods(from, to)` |

## Common Patterns

### Date Validation and Creation

```typescript
// Multiple ways to create dates
const date1 = d`20250115`;                    // Tagged template
const date2 = createYYYYMMDD("20250115");     // With validation
const date3 = parseToYYYYMMDD("2025-01-15");  // Parse format
const date4 = YYYYMMDDUtils.today();          // Today's date

// Type-safe validation
if (isYYYYMMDD(userInput)) {
  // userInput is now typed as YYYYMMDD
  const futureDate = d`${userInput}`.addDays(30);
}
```

### JSON Round-Trip

```typescript
// Parse from API
const response = await fetch('/api/transactions');
const data = await parseMoneyWorksResponse<Transaction[]>(response);

// Work with typed data
const filtered = data.filter(t => d`${t.transDate}`.gt(d`20250101`));

// Send back to API
const json = stringifyForAPI(filtered);
await fetch('/api/transactions', {
  method: 'POST',
  body: json
});
```

### Period Ranges

```typescript
// Generate fiscal year periods
const fiscalYear = PeriodUtils.range(202504, 202603);

// Process each period
fiscalYear.forEach(period => {
  const start = PeriodUtils.startDate(period);
  const end = PeriodUtils.endDate(period);
  console.log(`${PeriodUtils.format(period)}: ${start} - ${end}`);
});
```

## Best Practices

1. **Use Tagged Templates** - Prefer `d` and `p` for cleaner syntax
2. **Type Guards First** - Always validate untrusted input
3. **Leverage Fluent API** - Chain methods on YYYYMMDDWithMethods
4. **Configure JSON Parsing** - Use options for custom fields
5. **Namespace Imports** - Use utils namespaces for clarity
6. **Branded Types** - Prevent parameter mix-ups

## Quick Migration

From plain strings/numbers:
```typescript
// Before
const date = "20250115";
const period = 202501;

// After
const date: YYYYMMDD = createYYYYMMDD("20250115");
const period: Period = createPeriod(202501);

// Or with tagged templates
const date = d`20250115`;
const period = p`202501`;
```

From Date objects:
```typescript
// Before
const date = new Date();
const formatted = date.toISOString().split('T')[0].replace(/-/g, '');

// After
const date = d`${new Date()}`;
const formatted = date.toString(); // Already YYYYMMDD
```

From manual JSON parsing:
```typescript
// Before
const data = JSON.parse(jsonString);
data.transDate = data.transDate.replace(/-/g, '');

// After
const data = parseMoneyWorksJSON(jsonString);
// Dates automatically converted to YYYYMMDD
```
