# Getting Started with MoneyWorks Core

Welcome to MoneyWorks Core! This guide will help you set up the monorepo and get all components running. MoneyWorks Core is a comprehensive TypeScript SDK and application suite for MoneyWorks accounting software.

## What is MoneyWorks Core?

MoneyWorks Core is a monorepo containing multiple packages that work together:

- 🎯 **Utilities** - Core utilities and branded types
- 📚 **Canonical** - Pure MoneyWorks type definitions
- 💾 **Data** - Data access layer with smart field discovery
- 🖥️ **CLI** - Command-line tool for testing
- 🌐 **API** - REST API server with ElysiaJS
- 🎨 **Web** - Modern web application
- 📖 **Documentation** - This documentation site

## Quick Start (5 minutes)

### 1. Clone the Repository

```bash
# Clone the monorepo
git clone https://github.com/your-org/mw-core.git
cd mw-core

# Install dependencies for all packages
bun install
```

### 2. Set Up Environment Variables

Create a `.env` file in the root directory:

```bash
# MoneyWorks Connection
MW_HOST=http://localhost
MW_PORT=6700
MW_DATA_FILE=Acme Widgets.moneyworks
MW_USERNAME=your-username
MW_PASSWORD=your-password

# Optional: Folder authentication
MW_FOLDER_NAME=
MW_FOLDER_PASSWORD=

# API Server (optional)
API_PORT=3000
API_HOST=0.0.0.0

# Web App (for production)
VITE_CLERK_PUBLISHABLE_KEY=pk_test_...
DATABASE_URL=file:./data/app.db
ENCRYPTION_KEY=your-32-byte-encryption-key

# Development
NODE_ENV=development
```

### 3. Build All Packages

```bash
# Build all packages in dependency order
bun run build

# Or build specific packages
bun run build:utilities
bun run build:canonical
bun run build:data
```

### 4. Test Your Connection

Let's verify everything is working with the CLI:

```bash
# Run the CLI directly
bun run cli

# Or install globally for easier access
cd packages/cli
bun link
mw --help

# Test the connection
mw test-connection

# If successful, try exporting some data
mw export TaxRate --format=table
```

You should see output like:

## Running Different Components

### Command Line Interface (CLI)

The CLI is perfect for testing connections and exploring data:

```bash
cd packages/cli

# Development mode (auto-reload)
bun run dev

# Run commands
bun run cli export TaxRate
bun run cli eval "GetFieldValue('Company', 'Name')"
bun run cli shell  # Interactive shell
```

### REST API Server

Start the API server for web applications:

```bash
cd packages/api

# Development mode with auto-reload
bun run dev

# Production mode
bun run start
```

The API will be available at:
- Endpoints: `http://localhost:3000/api/v1`
- Swagger docs: `http://localhost:3000/api/v1/swagger`

### Web Application

Run the full-featured web interface:

```bash
cd packages/web1

# Development mode
bun run dev

# Build for production
bun run build
bun run start
```

Visit `http://localhost:5173` to see the web app.

### Documentation Site

You're reading it! To run locally:

```bash
cd packages/documentation

# Development mode
bun run dev

# Build for production
bun run build
```

## Project Structure

```
mw-core/
├── packages/

│   ├── canonical/       # MoneyWorks type definitions

│   ├── api/            # REST API server
│   ├── web1/           # Web application
│   └── documentation/  # This documentation site
├── scripts/            # Build and utility scripts
├── .env               # Environment variables (create this)
├── .env.example       # Template for environment variables
├── package.json       # Root package.json with workspace config
└── README.md          # Project overview
```

## Development Workflow

### 1. Making Changes

When developing, work in the appropriate package:

```bash
# Navigate to a package
cd packages/data

# Run tests in watch mode
bun test --watch

# Type checking
bun run typecheck

# Linting
bun run lint
```

### 2. Running Tests

Each package has its own test suite:

```bash
# Run all tests
bun test

# Run tests for a specific package
cd packages/utilities
bun test

# Run with coverage
bun test --coverage
```

### 3. Building Packages

Packages must be built in dependency order:

```bash
# Build everything
bun run build

# Build a specific package and its dependencies
cd packages/data
bun run build
```

## Common Tasks

### Connecting to Different MoneyWorks Servers

Update your `.env` file or create environment-specific files:

```bash
# .env.development
MW_HOST=http://localhost
MW_PORT=6700

# .env.production
MW_HOST=https://mw.company.com
MW_PORT=443
```

### Working with Multiple Data Files

The CLI supports multiple configurations:

```bash
# Create different config files
cp .env .env.company1
cp .env .env.company2

# Use specific config
MW_CONFIG=.env.company1 mw export Account
MW_CONFIG=.env.company2 mw export Account
```

### Debugging Connection Issues

<details>
<summary>Error: "Connection refused" or timeout</summary>

1. **Verify MoneyWorks Data Centre is running**
   ```bash
   # Check if the service is running
   ps aux | grep "MoneyWorks Data Centre"
   
   # Test network connectivity
   curl -I http://localhost:6700
   ```

2. **Check firewall settings**
   - Ensure port 6700 (or your custom port) is open
   - Check both system firewall and MoneyWorks security settings

3. **Verify the data file path**
   - The path must match exactly (case-sensitive)
   - Include the `.moneyworks` extension
</details>

<details>
<summary>Error: "Invalid username or password"</summary>

1. **Double-check credentials**
   - Ensure no trailing spaces in username/password
   - Verify caps lock is off

2. **Test with MoneyWorks client**
   - Try logging in with the official MoneyWorks client
   - Verify the user has appropriate permissions

3. **Check folder authentication**
   - If using folder auth, ensure both folder name and password are correct
</details>

<details>
<summary>Build errors or TypeScript issues</summary>

1. **Clean and rebuild**
   ```bash
   # Clean all build artifacts
   bun run clean
   
   # Reinstall dependencies
   rm -rf node_modules bun.lockb
   bun install
   
   # Rebuild everything
   bun run build
   ```

2. **Check TypeScript version**
   ```bash
   bun pm ls typescript
   # Should be 5.5.3 or later
   ```
</details>

## Environment-Specific Setup

### Development

For local development with hot reload:

```bash
# Start all development servers
bun run dev:all

# Or start specific services
bun run dev:api    # API server
bun run dev:web    # Web app
bun run dev:docs   # Documentation
```

### Production

For production deployment:

```bash
# Build all packages
bun run build

# Run production servers
NODE_ENV=production bun run start:api
NODE_ENV=production bun run start:web
```

### Docker

Using Docker for containerized deployment:

```bash
# Build the image
docker build -t moneyworks-core .

# Run with environment variables
docker run -p 3000:3000 \
  -e MW_HOST=host.docker.internal \
  -e MW_PORT=6700 \
  moneyworks-core
```
