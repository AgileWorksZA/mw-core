# OpenAPI Libraries Documentation

## Libraries Used for OpenAPI

**Last updated: 2025-05-01**

### openapi-typescript

The `openapi-typescript` library is used to generate TypeScript types from OpenAPI schemas and build type-safe API clients. It provides tools for:

1. Converting OpenAPI schemas to TypeScript types
2. Creating type-safe API clients
3. Integrating with React Query and SWR for data fetching

### openapi-types

The `openapi-types` package provides TypeScript definitions for OpenAPI specifications. It includes:

- Types for OpenAPI 2.0 (Swagger)
- Types for OpenAPI 3.0 
- Type definitions for request bodies, responses, parameters, etc.

In the Data Grid Designer component, we use these types:
- `OpenAPIV3.Document`: Represents the entire OpenAPI 3.0 document
- `OpenAPIV3.PathsObject`: Represents the paths in the OpenAPI document
- `OpenAPIV3.OperationObject`: Represents an API operation (e.g., GET, POST)
- `OpenAPIV3.ResponseObject`: Represents an API response

### Usage Examples

#### Creating a Client with openapi-typescript
```typescript
import createClient from "openapi-fetch";
import type { paths } from "./my-openapi-3-schema"; // generated by openapi-typescript

const client = createClient<paths>({ baseUrl: "https://myapi.dev/v1/" });

const {
  data, // only present if 2XX response
  error, // only present if 4XX or 5XX response
} = await client.GET("/blogposts/{post_id}", {
  params: {
    path: { post_id: "123" },
  },
});
```

#### Using openapi-types for OpenAPI Documents
```typescript
import { OpenAPIV3 } from "openapi-types";

function parseOpenAPI(document: OpenAPIV3.Document) {
  // Access paths
  for (const path in document.paths) {
    const pathItem = document.paths[path];
    
    // Access GET operations
    if (pathItem && pathItem.get) {
      const operation = pathItem.get;
      console.log(operation.summary);
      
      // Access responses
      const okResponse = operation.responses?.["200"];
      if (okResponse && "content" in okResponse) {
        const schema = okResponse.content?.["application/json"]?.schema;
        // Process schema...
      }
    }
  }
}
```

## Data Grid Designer Component

**Last updated: 2025-05-01**

The Data Grid Designer is a powerful component for visualizing and managing API data from OpenAPI/Swagger specifications. It automates the process of configuring TanStack Tables based on API schema definitions.

### Component Structure

The Designer component has been refactored into a modular architecture:

```
/designer/
  ├── designer.tsx           # Main export and layout
  ├── layout.tsx          # Layout component with sidebar integration
  ├── components/         # UI Components
  │   ├── api-loader.tsx   # URL input and API spec loading 
  │   ├── method-selector.tsx # API endpoint selection
  │   ├── api-documentation.tsx # Displays API documentation
  │   ├── data-path-adapter.tsx # Configure data and pagination paths
  │   ├── schema-properties.tsx # Displays schema properties
  │   ├── raw-schema.tsx   # Shows raw schema JSON
  │   ├── debug-panel.tsx  # Debug tools for state inspection
  │   ├── log-viewer.tsx   # Component for viewing log messages
  │   ├── empty-state.tsx  # Empty state placeholder
  │   ├── column-editor.tsx # Column configuration component
  ├── hooks/              # Custom hooks
  │   ├── use-designer-logs.tsx  # Log management
  │   ├── use-open-api-loader.ts  # API spec loading
  │   ├── use-property-filter.ts # Schema property filtering
  │   ├── use-schema-processing.ts # Schema processing
  ├── utils/              # Utility functions
  │   ├── schema/         # Schema processing utilities
  │   ├── openapi.ts      # OpenAPI parsing utilities
  └── types/              # TypeScript type definitions
      ├── schema.context.ts  # Schema-related types
      ├── open-api.context.ts # OpenAPI-specific types
```

### Key Features

1. **OpenAPI/Swagger Schema Parsing**: Automatically loads and parses API specifications
2. **Schema Property Extraction**: Extracts relevant properties from complex schemas
3. **Data Path Configuration**: Allows targeting specific paths in the API response
4. **Column Editor**: Configures TanStack Table columns based on schema properties
5. **Schema-Aware Type Handling**: Preserves type information from the API schema
6. **Calculated Columns**: Supports custom JavaScript formula expressions
7. **Debug Tools**: Provides debugging capabilities for state inspection
8. **Tab-Based UI**: Organizes functionality in a clean, tabbed interface
9. **Responsive Layout**: Resizable panels for better workspace organization

### Usage

```typescript
import { Designer } from "~/components/datagrid/designer";

export default function TableDesignerRoute() {
  return (
    <div className="w-full h-full overflow-hidden">
      <Designer />
    </div>

```

### State Management

The Designer uses Zustand-based state management through the `DesignerContext`:

```typescript
export interface IDesignerProps {
  url: string;
  documentation?: OpenAPIV3.Document;
  method?: string;
  schema?: OpenAPIV3.SchemaObject | OpenAPIV3.ReferenceObject;
  properties?: SchemaProperties;
  paths: {
    data: string;
    pagination: string;
  };
  columns: ColumnDef<unknown>[];
}

export const DesignerContext = createStoreWithProducer(create, {
  context: DefaultContext,
  on: {
    reset: () => rawReturn(DefaultContext),
    update: (context, event) => ({ ...context, ...event.update }),
  },
});
```

### Column Configuration

Columns are configured with proper metadata based on schema properties:

```typescript
// Example column with schema metadata
const column: ColumnDef<unknown> = {
  id: propertyKey,
  header: propertyKey.charAt(0).toUpperCase() + propertyKey.slice(1),
  accessorKey: propertyKey,
  enableSorting: true,
  enableColumnFilter: true,
  meta: {
    type: "string",
    required: true,
    nullable: false,
    description: "User's full name"
  },

```
